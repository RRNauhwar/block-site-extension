    Strict mode & stronger protection

    Focus Lock blocks sites and locks itself during a session. However, be aware of an important limitation:

    A Chrome extension cannot prevent its own removal. Anyone with access to the browser can open chrome://extensions and disable or uninstall Focus Lock, which stops all blocking. This is a deliberate security boundary in Chrome — no extension can override it.

    How to make blocking truly enforced

    For real, tamper-resistant enforcement (e.g. for schools, parents, or serious self-control), combine Focus Lock with OS- or enterprise-level policies that lock the extension in place:

    1. Force-install & lock via enterprise policy. Use Chrome's ExtensionInstallForcelist policy to force-install the extension so it cannot be removed by the user, and ExtensionSettings with "installation_mode": "force_installed" to prevent disabling it.
    2. Managed Chrome / Google Admin console. On managed Chromebooks or enrolled browsers, an administrator can pin the extension and block access to chrome://extensions entirely.
    3. Separate OS user account. Create a restricted, non-admin operating-system user for study time. Without admin rights, the policies above cannot be changed.
    4. ChromeOS / parental controls. On ChromeOS or with Family Link, a supervising account can enforce extension and site policies that the supervised user cannot undo.
    Chrome Enterprise policy reference

    Without these OS/enterprise measures, treat Focus Lock as a strong deterrent and accountability tool — not an unbreakable lock.

    How Focus Lock works

    • Blocking uses Chrome's declarativeNetRequest dynamic rules.
    • Blocked pages are redirected to a local blocked screen.
    • Timer state is saved in chrome.storage.local, so it survives a browser restart.
    • Once a session starts, there is no early unlock — blocking stays on until the timer ends.
    • The block list and timer are locked while a session is active.
